# Быстрый старт edt-bridge

edt-bridge даёт AI-агенту (Claude Code и другим MCP-клиентам) доступ к **живой модели 1C:EDT**:
чтение метаданных и BSL, валидация запросов, Синтакс-помощник платформы, а также запись –
создание объектов и расширений, рефакторинг, сборка `.epf`/`.erf` и обновление информационных
баз, всё через штатный движок EDT.

Из чего он состоит:

- **плагин 1C:EDT** (jar в `dropins/`) – поднимает MCP-сервер внутри EDT на
  `http://127.0.0.1:8770/mcp`;
- **обвязка `edt-bridge-mcp`** (Python, ставится через pipx) – stdio MCP-сервер, с которым
  разговаривает клиент; пробрасывает запросы в EDT, сама поднимает headless EDT, если ни одна не
  открыта, и сама доставляет jar плагина, если его нет.

## Требования

- **1C:EDT** установлена (её headless-запуск обвязка сделает сама; либо держите открытой GUI EDT
  с вашим проектом).
- **Python 3.10+** и **pipx** (см. ниже, если pipx нет).
- Только для сборки `.epf`/`.erf` и обновления ИБ: установленная **платформа 1С:Предприятие**,
  совместимая с версией проекта.

## Установка (обычная – через pipx)

```bash
pipx install edt-bridge-mcp
```

Если pipx нет:

```bash
python -m pip install --user pipx
python -m pipx ensurepath      # затем переоткрыть терминал
```

macOS: `brew install pipx && pipx ensurepath`.

## Подключение к Claude Code

```bash
claude mcp add edt-bridge -- edt-bridge-mcp --workspace "D:\\путь\\к\\workspace-EDT"
```

`--workspace` – папка рабочей области EDT (с `.metadata`), которую обвязка поднимет в headless,
если EDT не запущена. При открытой GUI EDT с плагином обвязка просто подключится к ней.

Для write-инструментов (создание/рефакторинг/сборка) задайте токен – тогда запись включена, а не
только dry-run:

```bash
claude mcp add edt-bridge --env EDT_BRIDGE_TOKEN=<любая-строка> -- \
  edt-bridge-mcp --workspace "D:\\путь\\к\\workspace-EDT"
```

## Проверка

В сессии Claude Code попросите список проектов или справку по API – например: "покажи открытые
проекты EDT" (инструмент `edt_projects`) или "найди в Синтакс-помощнике ТаблицаЗначений.Добавить"
(`edt_platform_help`). Первый вызов может подниматься до нескольких минут, если обвязка стартует
headless EDT и грузит модель; далее – мгновенно.

Ручная проверка сервера (когда EDT/headless уже поднят):

```bash
curl -s http://127.0.0.1:8770/status
```

Дашборд со статусом и запуском любого инструмента – `http://127.0.0.1:8770/` в браузере.

## Обновление

```bash
edt-bridge-mcp self-update             # обновить и jar плагина (GitHub Releases), и обвязку (PyPI)
edt-bridge-mcp self-update --jar-only  # только jar
```

Старые jar в `dropins/` при этом сносятся сами (два jar заставили бы EDT загрузить произвольный).
Запущенная EDT держит старый код до перезапуска.

## Установка вручную (без обвязки)

Если хотите запускать плагин сами: возьмите jar со
[страницы Releases](https://github.com/keyfire/edt-bridge/releases), положите **один** в `dropins/`
EDT, перезапустите EDT – сервер поднимется на 8770 (или следующем свободном порту). Клиент можно
подключить напрямую по HTTP: `{ "edt-bridge": { "type": "http", "url": "http://127.0.0.1:8770/mcp" } }`.
Подробности и сборка из исходников – в [README.ru.md](README.ru.md).

## Если что-то не так

- **Клиент не видит инструменты сразу после старта** – обвязка поднимает headless EDT в фоне;
  список инструментов появится, когда модель загрузится (приходит `tools/list_changed`).
- **"a GUI EDT is running"** – открыта GUI EDT без плагина, и обвязка не трогает её (держит замок
  рабочей области). jar она в `dropins/` доставит – перезапустите эту EDT, чтобы активировать мост,
  либо закройте её.
- **Порт 8770 занят** – плагин берёт следующий свободный, обвязка находит его сканированием;
  можно задать явно через `EDT_BRIDGE_PORT`.
- **write-инструмент отказывает "requires a configured token"** – задайте `EDT_BRIDGE_TOKEN`
  (см. выше).
- **`edt_dump_external_object` не собирает** – нужна установленная платформа 1С:Предприятие,
  совместимая с версией проекта.
