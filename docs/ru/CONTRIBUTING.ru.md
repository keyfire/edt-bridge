# Участие в edt-bridge

Спасибо за интерес. edt-bridge – это плагин для 1C:EDT плюс небольшая Python-обвязка; здесь описаны
сборка, соглашения и как предлагать изменения.

## Базовые правила

- **Clean-room.** edt-bridge – независимая реализация, написанная с нуля (см. [ORIGIN.md](../../ORIGIN.md)).
  Присылайте только собственный код, построенный на открытой спецификации MCP и публичных API
  плагинов Eclipse / 1C:EDT (`org.eclipse.*`, `com._1c.g5.v8.dt.*`). Не вставляйте декомпилированный
  или иной проприетарный код из EDT или другого инструмента.
- **Область.** Плагин отдаёт *живую* модель EDT по MCP. Новый инструмент должен требовать того, чего
  не даёт статический парсер (валидация работающей IDE, реальные типы, семантические ссылки, штатный
  рефакторинг / сборка). Если нет – standalone-инструмент подойдёт лучше.
- **Лицензия.** Присылая вклад, вы соглашаетесь, что он лицензируется под [Apache-2.0](../../LICENSE)
  (inbound = outbound).

## Сборка

Нужны **JDK 17+** и локальный **пул бандлов 1C:EDT** – плагин компилируется против собственных
бандлов EDT, которые проприетарны и берутся из вашей локальной установки EDT (поэтому сетевой
сборки нет).

```powershell
# Windows – по умолчанию: -Pool %USERPROFILE%\.p2\pool\plugins, -JdkHome %JAVA_HOME%
powershell -ExecutionPolicy Bypass -File scripts/build-nomaven.ps1
```

```bash
# macOS / Linux – --pool определяется автоматически из установленного компонента 1C:EDT
./scripts/build-nomaven.sh
```

Jar кладётся в `build/`. Для полной сборки доступен Maven + Tycho (`mvn -f pom.xml clean verify`) –
сначала отредактируйте `edt-bridge.target`. Подробнее: [README](README.ru.md#сборка-из-исходников).

## Запуск своей сборки

Скопируйте собранный jar в `dropins/` EDT (держите там ровно один jar edt-bridge – два заставят
Equinox грузить произвольный) и перезапустите EDT, либо дайте это сделать обвязке. MCP-сервер
поднимается на `http://127.0.0.1:8770/mcp`; дашборд на `http://127.0.0.1:8770/` содержит запускалку
для каждого инструмента. См. [Установку вручную](README.ru.md#установка-вручную-без-обвязки).

## Проверка изменения

Офлайн-набора тестов для Java-плагина нет – он проверяется против **живой EDT**: собрать, положить в
`dropins/`, перезапустить EDT и прогнать затронутые инструменты (через дашборд или MCP-клиент). Для
инструментов записи проверьте план dry-run (`apply=false`) перед `apply=true`. Тесты Python-обвязки
лежат в [`python/`](../../python/).

## Стиль кода

- **Java 17**, стандартные соглашения Eclipse / OSGi. Весь доступ к модели IDE держите в классах-
  гейтвеях в `...edtbridge.edt` – изменение API EDT должно оставаться там.
- **Английский** для всего кода, комментариев и идентификаторов.
- Один инструмент – один класс в `...edtbridge.tools`, регистрируется в `McpServer`.
- Имена инструментов – `edt_*` (snake_case); параметры – camelCase. Имена должны быть
  самоописательными – MCP-хост показывает инструменты всех серверов одним списком (см. README).

## Коммиты и pull request'ы

- Небольшие сфокусированные коммиты с ясным заголовком.
- Открывайте pull request в `main` и заполняйте шаблон (что изменилось, как проверено).
- Релизы режет мейнтейнер из локально собранного jar (CI не компилирует плагин), поэтому влитое
  изменение выходит в следующем теговом релизе.
