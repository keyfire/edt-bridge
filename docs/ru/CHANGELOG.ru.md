# История изменений

Все заметные изменения edt-bridge фиксируются здесь.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), проект следует
[семантическому версионированию](https://semver.org/lang/ru/). Jar плагина и обвязка
`edt-bridge-mcp` используют один номер версии.

## [Не выпущено]

## [0.3.0] – 2026-07-19

### Добавлено
- `edt_platform_installations` – показывает установки платформы 1С:Предприятие, которые EDT
  резолвит (для сборки бинарников и создания баз), включая полные установки, найденные на диске.
- `edt_create_infobase` – создание пустой файловой базы через EDT, с фоллбеком на поиск подходящей
  полной установки платформы на диске, если у EDT её нет.
- `edt_register_platform` – регистрация установки платформы в EDT.
- `edt_build_extension` – сборка расширения конфигурации (`.cfe`) через `ibcmd`: экспорт проекта
  средствами EDT и упаковка во временной базе, которая затем удаляется.
- Страница настроек EDT-Bridge (токен, порт MCP-сервера и переключатель evaluate), чтобы GUI-EDT,
  запущенная из обычного ярлыка, могла аутентифицировать инструменты записи.
- Раздел "Ошибки и обратная связь" в README; workflow trusted publishing для PyPI.

### Исправлено
- `edt_infobases` теперь разворачивает группы баз – груповые/серверные базы больше не теряются.

### Изменено
- Внутреннее: единый модельный шлюз разбит на сфокусированные гейтвеи по областям (проекты, чтение
  и запись метаданных, формы, платформа, отладка, BSL, документация) плюс общие хелперы. Чистый
  рефакторинг – поведение не меняется.

## [0.2.1] – 2026-07-18

### Изменено
- Сплошная типографика по стилю проекта (среднее тире вместо длинного, три точки вместо символа
  многоточия), включая описания инструментов и Python-обвязку.

## [0.2.0] – 2026-07-18

### Добавлено
- `edt_platform_help` – Синтакс-помощник платформы 1С:Предприятие из поставки EDT (реальная справка
  API, с поиском, рус + англ).
- Многопоточный MCP-сервер (ограниченный пул воркеров).
- Фоллбек порта: если 8770 занят, берётся следующий свободный, и обвязка его находит.

### Исправлено
- `apply` создания проектов теперь работает надёжно (строится и привязывается свежий детаченный
  Configuration / внешняя обработка, а не роняется жизненный цикл headless-EDT).

## [0.1.1] – 2026-07-18

### Добавлено
- Инструменты записи цикла создать / разработать / собрать / доставить (по умолчанию dry-run).

### Исправлено
- `apply` создания проектов приведён к жизненному циклу headless-EDT.

## [0.1.0] – 2026-07-18

### Добавлено
- Полный цикл создать / разработать / собрать / доставить, открытый по MCP.
- Stdio-обвязка `edt-bridge-mcp` (pipx): пробрасывает запросы в запущенную EDT, сама поднимает
  headless-EDT, когда ни одна не открыта, и доставляет jar плагина в `dropins/` EDT.

## [0.0.1] – 2026-07-11

### Добавлено
- Первый выпуск: инструменты чтения по MCP (проекты, ошибки валидации, свойства и перечень
  метаданных, ссылки, валидация запросов, формы), встроенный дашборд и localhost MCP-сервер.
